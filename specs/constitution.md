# 📜 Constitution: Carmen Sandiego Game

This document defines the core principles, technology stack, and development guidelines for the Carmen Sandiego project.

## 🚀 Core Mission
To build a modern, high-fidelity investigation game that pays homage to the original Carmen Sandiego series while utilizing modern web technologies and AI-driven content generation.

## 🛠️ Technology Stack
- **Frontend**: React 18, TypeScript 4.9+, Styled Components.
- **Backend**: Express.js, Node.js.
- **Database**: MongoDB (Mongoose).
- **Tooling**: Spec Kit (SDD), Vite (or React Scripts), Concurrent execution for dev.

## 📏 Development Rules
- **TypeScript First**: Ensure strict typing across all components and API routes.
- **Component Architecture**: Use functional components and hooks. Styles should be co-located or in a dedicated `styles.ts` using Styled Components.
- **Responsive Design**: prioritize a "Travel Terminal" UX that works on both desktop and mobile.
- **Parity**: Game logic should aim for 100% parity with original game mechanics (clue generation, travel restrictions, dossier management).
- **Specifications (SDD)**: All major features must start with a `/specify` command in Spec Kit.

## 📂 Project Structure
- `src/`: Frontend React source.
- `api/`: Backend Express routes and logic.
- `specs/`: Spec Kit documentation and feature plans.
- `public/`: Static assets.

## 🐙 Git Workflow
- All new features should be developed on a branch following the `feat/###-name` pattern.
- Commits should be descriptive and auto-generated by Spec Kit where possible.
